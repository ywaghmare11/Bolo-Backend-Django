## 2026-07-19

- `[BE]` Phase 2 complete: `common` app foundation, OTP→JWT auth, and a core Task lifecycle vertical slice — the first real API surface in this repo (every app previously had only `models.py`/`admin.py`, zero serializers/services/repositories/views/urls anywhere). Scope deliberately narrowed to core lifecycle only (create/list/detail/edit/delete + accept/done-a/done-d/cancel/remind, minimal label create/list); subtasks, comments, evidence, voice recording, full label CRUD, and audit logging are all still Phase 3+.
  - **`apps/common`:** `responses.py` (`success_response`/`failure_response`), `exceptions.py` (`AppError` + 4 subclasses — extended with an optional `data` field beyond the literal guidelines.md text, needed for `verify-otp`'s documented `data.attemptsRemaining`), `pagination.py` (`BoloPageNumberPagination` matching the `{page,limit,total}` envelope), `permissions.py`, `email.py` (`EmailService`, console backend in dev/test, SES is a Phase 3+ swap). `config/exception_handler.py` wired as DRF's `EXCEPTION_HANDLER`.
  - **`apps/auth`:** OTP request/verify/logout, plus a **deliberate deviation from `docs/ops/security.md`'s locked W1 decision** — added real access+refresh token handling (user-requested, not in the original contract): 15-min JWT access token (`token` cookie, via `djangorestframework-simplejwt`'s `AccessToken` used as a bare encode/decode primitive) + a rotating opaque refresh token (`refresh_token` cookie, new `RefreshToken` model/migration, SHA-256-hashed at rest like `OtpCode`) with reuse-detection (replaying a revoked refresh token revokes every token for that user). `docs/ops/security.md`'s Authentication section rewritten to match. Custom `CookieJWTAuthentication` resolves `request.user` straight to `apps.users.User` (no `AUTH_USER_MODEL` involved) — required adding `is_authenticated`/`is_anonymous` class attributes to `User` since it doesn't extend `AbstractUser`/`AbstractBaseUser` and DRF's `IsAuthenticated` permission expects them (caught via manual curl testing, not by `manage.py check`).
  - **`apps/tasks`:** full core-lifecycle vertical slice wired through `apps/labels` (minimal create/list) and a new `apps/notifications::dispatch_notification()` helper (the only place `Notification` rows get written, per Architecture Rule 7). `TaskRepository.delete()` explicitly clears children in a transaction rather than relying on the `Comment`/`Evidence`/`parent_task` `PROTECT` FKs staying vacuously empty — costs nothing now, means zero rework once Phase 3 adds those creation endpoints. List/detail responses always include the documented nested keys (`subtasks`/`comments`/`evidence`/`voiceRecording`/counts) as real (currently always-empty) queries, not omitted or hardcoded.
  - **Testing:** hit a real pytest 9.1.1 / pytest-django 4.12.0 incompatibility (`AssertionError` in `_pytest/fixtures.py` on every single `@pytest.mark.django_db` test, reproduced with zero custom fixtures) — pytest-django 4.12.0 is the latest release and doesn't support pytest 9's fixture-finalizer internals yet. Pinned `pytest==8.4.2` in `requirements/dev.txt`/`test.txt` instead of waiting on an upstream fix. Also needed `ALTER USER bolo_django CREATEDB` on the local Postgres role (pytest-django creates/destroys a throwaway `test_bolo_django` DB per run) — noted in `README.md`. 22 tests added across `apps/{auth,labels,tasks}/tests/`, all passing; `ruff check` clean on every new/edited file (pre-existing lint findings in untouched apps' scaffold stubs left alone).
  - Full manual curl walkthrough (OTP → verify → refresh-rotation-and-reuse-detection → label create → task create/accept/done-a/done-d/cancel/delete/remind, cross-role 403s, `TASK_TERMINAL` 409) run against `manage.py runserver` and a seeded tenant — all passed; seed data cleaned up afterward.
